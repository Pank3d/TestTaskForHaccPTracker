# HACCP Tracker - Справочник сотрудников

Приложение для управления справочником сотрудников предприятий.

## Технологии

- React 19
- TypeScript
- Material UI v6
- React Router v6
- TanStack Query (React Query)
- Zod (валидация форм)
- Zustand (управление состоянием)
- FSD (Feature-Sliced Design) архитектура

## Запуск проекта

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Укажите URL API в файле `.env`:

```env
VITE_API_URL=http://your-api-url.com/api
```

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

## Функциональность

- Авторизация через JWT
- Просмотр списка сотрудников
- Создание, редактирование и удаление сотрудников
- Фильтрация и сортировка таблицы
- Переключение между предприятиями
- Валидация форм

## Структура проекта

Проект организован по методологии FSD (Feature-Sliced Design):

```
src/
├── app/          # Конфигурация приложения, роутинг
├── pages/        # Страницы приложения
├── widgets/      # Сложные компоненты (Sidebar, LoginWidget, etc.)
├── features/     # Бизнес-фичи (модалки, формы, таблицы)
├── entities/     # Бизнес-сущности (user, staffer, enterprise)
└── shared/       # Переиспользуемые компоненты и утилиты
```

## Архитектура

### FSD (Feature-Sliced Design)

Проект следует принципам FSD - методологии архитектурного дизайна для фронтенд приложений:

- **Слои** организованы по уровню ответственности (app → pages → widgets → features → entities → shared)
- **Слайсы** группируют код по бизнес-доменам (user, staffer, enterprise)
- **Сегменты** разделяют код по техническому назначению (ui, model, api, config)

### Основные паттерны

- **Composition over Configuration** - компоненты собираются из мелких переиспользуемых частей
- **Controlled Components** - формы полностью контролируются через React state
- **Custom Hooks** - бизнес-логика изолирована в переиспользуемых хуках
- **React Query** - серверное состояние управляется декларативно
